---

## 一、项目概述

本系统是一个**双碳（碳达峰、碳中和）合规审查平台**，帮助企业检查项目是否符合国家和地方的能耗与碳排放政策，并提供基于 AI 的政策智能问答。

**核心功能模块**：用户认证、政策管理、合规审查、AI 智能问答、管理后台、操作日志

### 2. API 响应格式规范

**成功响应**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

**失败响应**：
```json
{
  "code": 400,
  "message": "具体错误原因",
  "data": null
}
```

**标准 HTTP 状态码**：
- `200`：成功
- `400`：请求参数错误
- `401`：未登录
- `403`：无权限
- `404`：资源不存在
- `500`：服务器内部错误

### 3. 错误处理规范

```python
# 所有 API 接口必须有 try/except
@auth_bp.route('/login', methods=['POST'])
def login():
    try:
        data = request.get_json()
        # ... 业务逻辑
        return jsonify({'code': 200, 'message': '登录成功', 'data': ...})
    except ValueError as e:
        return jsonify({'code': 400, 'message': str(e)}), 400
    except Exception as e:
        app.logger.error(f'Login error: {e}')
        return jsonify({'code': 500, 'message': '服务器内部错误'}), 500
```

# 使用上下文管理器，确保 session 正确关闭
with app.app_context():

## 十一、关键技术难点说明

### 11.1 RAG 向量化流程

```
上传政策文件（PDF/Word）
  ↓
document_processor.py 解析文本，按段落切分
  ↓
每段生成一个向量（通过 DeepSeek Embedding API）
  ↓
存入 FAISS 向量数据库（本地文件持久化）
  ↓
用户提问时，问题也转为向量，在 FAISS 中相似度检索
  ↓
检索到的相关条款 + 用户问题 → 一起发给 DeepSeek 生成回答
```

### 11.2 合规审查引擎流程

```
用户提交项目信息（JSON）
  ↓
compliance_engine.py 根据项目行业、地区，筛选相关政策
  ↓
对每条政策的每个条款，检查项目是否合规
  ↓
生成 review_details 记录（每项 pass/fail/warning）
  ↓
计算合规得分（compliance_score）
  ↓
调用 report_generator.py 生成 Word/PDF 报告
```

### 11.3 需要特别注意的地方

| 问题 | 说明 | 建议方案 |
|------|------|---------|
| **DeepSeek API 限流** | 免费 Key 有每秒请求数限制 | 添加重试机制 + 请求队列 |
| **FAISS 索引并发** | 多用户同时查询可能冲突 | 加文件锁或改用向量数据库服务 |
| **大文件上传超时** | PDF 解析可能耗时较长 | 改为异步任务（Celery） |
| **密码当前明文** | `models.py` 中密码未加密 | **最高优先级修复** |
| **SQL 注入防护** | 使用 SQLAlchemy ORM 已防护 | 注意不要用原生 SQL 拼接 |
