## 一、项目概述

本系统是一个**双碳（碳达峰、碳中和）合规审查平台**，帮助企业检查项目是否符合国家和地方的能耗与碳排放政策，并提供基于 AI 的政策智能问答。

**核心功能模块**：用户认证、政策管理、合规审查、AI 智能问答、管理后台

### 4.1 认证模块 `/api/auth/`

| 接口 | 方法 | 路径 | 状态 | 说明 |
|------|------|------|------|------|
| 登录 | POST | `/api/auth/login` | ✅ | 用户名 + 密码登录 |
| 登出 | POST | `/api/auth/logout` | ✅ | 清除 session |
| 当前用户 | GET | `/api/auth/me` | ✅ | 获取当前登录用户信息 |

#### 登录接口详情

**请求**：
```json
POST /api/auth/login
{
  "username": "admin",
  "password": "admin123"
}
```

**成功响应**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "id": 1,
    "username": "admin",
    "real_name": "管理员",
    "role": "admin"
  }
}
```

**失败响应**：
```json
{
  "code": 401,
  "message": "用户名或密码错误"
}
```

#### 当前用户接口详情

**请求**：
```
GET /api/auth/me
（无需参数，通过 session 识别用户）
```

**成功响应**：
```json
{
  "code": 200,
  "data": {
    "id": 1,
    "username": "admin",
    "real_name": "管理员",
    "role": "admin",
    "email": "admin@example.com",
    "phone": "13800138000"
  }
}
```

**未登录响应**：
```json
{
  "code": 401,
  "message": "请先登录"
}
```

## 六、Mock 数据说明

对于标注为 🔨 或 📋 的后端待开发接口，前端需要先用 Mock 数据开发页面。

### Mock 方式

在 `src/api/index.ts` 中，可以为未就绪的接口写 Mock 函数：

```typescript
// api/index.ts 示例

// 已就绪的接口 — 直接调用后端
export const authApi = {
  login: (username: string, password: string) =>
    api.post('/auth/login', { username, password }),
  me: () => api.get('/auth/me'),
}

// 未就绪的接口 — 临时 Mock
const useMock = true // 后端完成后改为 false

export const userApi = {
  list: () => {
    if (useMock) {
      return Promise.resolve({
        data: {
          code: 200,
          data: {
            users: [
              { id: 1, username: 'admin', role: 'admin', status: 'active' },
              { id: 2, username: 'user1', role: 'user', status: 'active' },
            ],
            total: 2
          }
        }
      })
    }
    return api.get('/admin/users')
  }
}
```

### 需要前端 Mock 的接口

| 接口 | Mock 建议返回值 |
|------|----------------|
| 用户注册 `POST /api/auth/register` | `{"code": 200, "message": "注册成功"}` |
| 问答反馈 `POST /api/qa/feedback` | `{"code": 200, "message": "感谢反馈"}` |
| 仪表盘统计 `GET /api/admin/stats` | 总政策数、总审查数、合规率等 |
| 用户列表 `GET /api/admin/users` | 用户名、角色、状态、创建时间 |
| 系统设置 `GET/PUT /api/admin/config` | RAG 参数、系统名称等配置项 |

```
style: 样式调整（不影响逻辑）
docs: 文档变更
refactor: 代码重构（不改变功能）

示例：
feat(login): 实现登录页面
fix(review): 修复审查结果不显示的问题
style(home): 调整首页布局
```
